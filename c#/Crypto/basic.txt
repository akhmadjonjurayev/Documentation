1. Hex (16 - lik sanoq tizimi)
 Hex belgilar: 0-9 va A-F
 Hex - uzun binary ning ixcham ko'rinishi
 Binary: 01001111
 Hex: 4F
 C# da 
 byte b = 79;
 string hex = b.ToString("X2"); // 4F
 byte[] bytes = {1, 2, 255};
 string hex = Convert.ToHexString(bytes); // 0102FF

2. Base64 - Binary data ni text ko'rinishida saqlash uchun. Binary datani tog'ridan to'g'ri textga qo'yib bo'lmaydi.
Binary data ichida:
  * 0x00 (NULL)
  * control characters
  * newline bo'ladi -> transport buziladi. X

Nima uchun base64 = 26 ta katta kichik harflar va 10 ta raqam va +,/ belgi = Jami 64 ta
Matematik sabab 2 ^ 6 = 64
Har bir Base64 belgisi 6 bit axborot saqlaydi. Nega 6 bit kompyuter olib yuradi.
1 byte = 8 bit Base64 1 belgi = 6 bit
3 byte = 24 bit
24 bit / 6 = 4 ta Base64 belgi. Shu sababli Base64 har doim 3 byte -> 4 belgi qiladi. Doim axborotni 6 blockli belgilarga bo'ladi

4. PKI (Public Key Infrastructure) - bu ishonch tizimi. Certificate orqali tekshiramiz. CA (Certificate Authory) orqali tasdiqlaymiz.
Asosiy komponentalar:
 I. Certificate. Bu Public key + Identity (kimligi) + Signature
 II. CA (Certificate Authory). Masalan : Let's Encrypt, DigiCert
 III. Root CA - Browzer (operatsion sistema) ichida oldindan o'rnatilgan bo'ladi: trusted root certificates.
 IV. Chain of Trust.
   Root CA -> Intermadiate CA -> Server Certificate.
  Sertificate ichida nimalar bor:
   * Subject  -> kim (example.com)
   * Issuer   -> kim berdi (CA)
   * Public Key -> RSA/ECC key
   * Validity -> amal qilish muddati
   * Signature -> CA imzosi

5. Nato'gri kalit bilan decrypt qilsa nima bo'ladi.
  5.1 Simmetrik shifrlash. Oddiy authentification yo'q shifrlashda xatolik bermaydi. Random text chiqaradi. Shifrlash algoritmi transformatsiya
qiladi. Natog'ri kalit -> natog'ri transformatsiya -> lekin bari bir input bor.
  Authenticated encryption (masalan: AES-GCM). Bu zamonaviy usul. Agar kalit nato'g'ri bo'lsa decryption fail beradi, exception chiqadi
yoki verify bo'lmaydi. Bu yerda nafaqat decrypt balki integrity check ham bor tag orqali.
  Faqat AES-CBC -> ishlatilsa silent corruption bo'lishi mumkin. AES-GCM ishlatilsa -> har doim xavfsiz (fail-fast)
  5.2 Assimetrik shifrlash (RSA, ECC). Natija xatolik chiqadi ayniqsa padding ishlatilsa (PKCS#1, OAEP). Bazi hollarda random data chiqishi mumkin
lekin u kam uchraydi.
