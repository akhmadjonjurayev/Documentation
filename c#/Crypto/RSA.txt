  Asymmetric shifrlash
Qoidalar:
  * Public key -> encrypt
  * Private key -> decrypt
  * Private key -> sign
  * Public key -> verify

RSA - matematik asosda.
 1. p, q  2 ta tub son olinadi.
 2. n = p * q
    f(n) = (p - 1)(q - 1)
 3. Public expontent tanlanadi. e
 4. Private exponent f(n) (Eyler funsiyasi siyasida) o'zaro tub son tanlanadi.
  d = e ^ -1 mod f(n)
  Public key -> (e, n)
  Private key -> (d, n)
  Encryption c = m ^ e mod(n)
  Decryption m = c ^ d mod(n)

RSA dagi xatolar:
  x Raw Rsa Engine ishlatish (padding yo'q)
  x 1024 bit ishlatish
  x Private key ni clientga yuborish
  x Katta data ni bevosita RSA bilan shifrlash

Assimetrik kriptografiya:
 * Kalit almashish
 * Gigital signature
 * Idenitity verification uchun ishlatiladi

RSA da tub sonlarni topishda Miller-Rabin testidan foydalaniladi. gcd(m, n) != 1 bo'lsa RSA buziladi.
Raw RSA nima
 c = m ^ e mod(n)
 m = c ^ d mod(n)
Hech qanday padding yo'q. Bu juda xavfli.

 * Deterministic, bir xil message -> bir xil cipher
 * Chosen - plaintext attack uchun
 * Small - exponent attack uchun

AsymmetricKeyParameter da nima bo'ladi. Bouncy Castle da RSA key classlari

 CRT parametrlari:
Normal decrypt: m = c ^ d mod(n)
CRT bilan:
  m1 = c ^ dP mod(p)
  m2 = c ^ dQ mod(q)
Chinese Remainder Theorem bilan birlashtiriladi.


Digital Signature - bu ma'lumotni kim yuborganini va u o'zgarmaganini isbotlaydigan kriptografik imzo. U 2 ta narsani ta'minlaydi:
Authentification kim yuborganini, Integrity ma'lumot o'zgarmaganini bilamiz.
Digital Signature ishlashi:
 1. Sender message ni hash qiladi
    message = "Salom"
    hash = SHA256(message)
 2. Hash private key bilan shifrlanadi
    signature = Encrypt(hash, privateKey)
    RSA bo'lsa
    signature = hash^d mod(n)
 3. Message + signature yuboriladi
 4. Receiver tekshiradi
    * message ni hash qiladi
    hash1 = SHA256(message)
    * signature ni public key bilan ochamiz
    hash2 = signature^e mod(n)
    * solishtiriladi 
    hash1 == hash2
    Agar o'zgarmagan bo'lsa message o'zgarmagan, haqiqiy yuboruvchi
